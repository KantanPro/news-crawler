# News Crawler — WooCommerce ダウンロード販売ガイド

v3.3.0 以降、プラグイン内のライセンスキー認証は廃止しています。販売・配布は **WooCommerce のダウンロード可能な商品** で行います。

## ライセンス（GPL）

- 配布物は **GPL v2 or later**（`license.txt` 参照）
- 有料販売は可能（サポート・更新・配布 ZIP の提供として）
- 購入者に渡したソースは GPL の下で再配布される可能性がある点は商品説明に明記すること

## WooCommerce 設定手順

### 1. 前提

- WooCommerce 有効化
- **WooCommerce → 設定 → 商品 → ダウンロード可能な商品** でダウンロードを有効化

### 2. 商品作成

| 項目 | 推奨 |
|------|------|
| 商品タイプ | シンプル商品 |
| バーチャル | はい |
| ダウンロード可能 | はい |
| ダウンロードファイル | `news-crawler.zip`（フォルダ名 `news-crawler` で zip 化） |
| ダウンロード制限 | 空欄（無制限）または 3〜5 回 |
| ダウンロード有効期限 | 空欄 または 365 日 |

ZIP の作り方（リポジトリ内）:

```bash
./create_release_zip.sh
```

配布用 ZIP には `license.txt` を含めます。

### 3. 購入後メール

- 注文完了メールにダウンロードリンクが含まれるよう **WooCommerce メール** を確認
- マイアカウント → 注文 からも再ダウンロード可能にする

### 4. 商品ページに書くこと（必須に近い）

- 対応 WordPress / PHP バージョン（プラグインヘッダーと同じ）
- 必要な外部 API（OpenAI / YouTube Data API / X Developer 等は**利用者自身のキー**）
- サーバー cron の設定が必要な旨（自動投稿）
- GPL に基づく再配布の可能性
- サポート範囲・返金条件

### 5. GitHub 公開リポジトリとの関係

公開リポジトリからも ZIP が取れる状態だと、有料版と無料取得が二重になります。

- **推奨**: 販売用 ZIP は WooCommerce のみ。GitHub は非公開にするか、Release を販売版より遅らせる
- 更新配布も WooCommerce 商品ファイルの差し替え + 購入者への案内メール

## 利用規約で明記する例

- 返金ポリシー（ダウンロード商品は原則不可が一般的）
